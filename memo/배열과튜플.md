## 튜플

각 인덱스에 정해진 타입이 있는 배열

```ts
let tuple: [string, number];
tuple = ["hello", 42]; // OK
tuple = [42, "hello"]; // ERROR
```

## as const

객체나 배열을 `as const`로 선언하면 모든 프로퍼티가 `readonly`가 되고, 리터럴 타입으로 고정된다.
그리고 배열의 경우 튜플 타입이 된다.

```ts
let tuple = [10, 20] as const;
// tuple[0] = 30; // ERROR: readonly이므로 변경 불가
// tuple.push(30); // ERROR: readonly이므로 변경 불가
```

이것을 const assertion이라고 부른다.

```ts
let xyz = "xyz" as const; // 'xyz' 리터럴 타입
// let도 as const를 사용할수는 있지만 값을 변경할 수 없으므로 별 의미가 없다.
xyz = "abc"; // ERROR
```

primitive 타입에 as const를 사용하면 리터럴 타입이 된다.
object 타입의 값 뒤에 as const를 사용하면 모든 프로퍼티가 readonly가 되고, 가능한 리터럴 타입으로 고정된다.

````ts
let obj = {
  name: "taeyeong",
  age: 30,
} as const;
// obj의 타입은 { readonly name: 'taeyeong'; readonly age: 30; }
```
````
